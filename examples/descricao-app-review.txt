AUTHENTICATION

The dashboard authenticates to Meta server-to-server using a System
User access token signed with appsecret_proof (HMAC-SHA256). There is
no end-user OAuth flow because end users are our own internal
employees who access the dashboard via [SSO / VPN / explain access
control]. The screencast includes captions explaining this so reviewers
can follow the auth model.

WHAT THE APP DOES

The dashboard (live at [APP_URL]) gives our internal media team a
single pane to:

1. View consolidated insights across the [N] accounts (spend,
   impressions, clicks, CTR, CPC, reach) with delta vs. previous
   period.
2. List all campaigns with status, daily/lifetime budget, and KPIs.
3. Pause or activate campaigns — every write requires explicit
   human approval (HITL) inside a dedicated confirmation panel.
4. Update daily budgets — same HITL approval flow, with server-side
   validation.
5. Review an append-only audit log of every API call (reads and
   writes) with operation, resource_id, before/after state, and
   timestamp.

WHY STANDARD ACCESS IS REQUIRED

Development Access (60 points/hour) is exhausted in seconds during
normal daily operations across [N] accounts and [M]+ active campaigns.
Standard Access (9,000 points/hour) is the minimum capacity required
to support the team's actual workflow without artificial throttling.

COMPLIANCE AND SAFETY

- Authentication via System User token + appsecret_proof
  (HMAC-SHA256), no client-side token exposure
- Append-only JSONL audit log of every call with before/after state
- Server-side rate limiter with circuit breaker on quota exhaustion
- HITL approval gate on every write — no automated/background writes
- HTTPS-only, secrets server-side, English UI throughout

Data accessed via this feature is used exclusively to manage our own
ad accounts. It is not resold, shared, or used for any other purpose.
